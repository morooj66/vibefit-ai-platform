# VibeFit AI Platform — Design System

> مرجع تصميمي لمرحلة MVP. يصف القرارات البصرية والمكوّنات والصفحات دون تنفيذ كود.  
> Design Tokens مُهيكلة للتحويل لاحقًا إلى **Tailwind CSS** (`theme.extend`).

---

## 1. شخصية وهوية العلامة (Brand Personality & Identity)

### 1.1 جوهر العلامة

| المحور | الوصف |
|--------|--------|
| **الشخصية** | ودودة، محفّزة، واضحة — ليست عدوانية ولا سريرية |
| **النبرة** | «مرشد رياضي بسيط» وليس «مدرب محترف صارم» |
| **الإحساس** | طاقة هادئة + تنظيم + ثقة |
| **الوعد البصري** | «رحلتك الرياضية تبدأ من هنا — خطوة بخطوة» |

### 1.2 سمات الهوية البصرية

- **بساطة**: واجهة نظيفة بعناصر قليلة لكل شاشة.
- **تدرج بصري واضح**: عنوان → محتوى → إجراء (CTA).
- **مساحات بيضاء سخية**: تقليل الإرهاق للمبتدئين.
- **لمسات طاقة**: لون Primary وAccent في النقاط الحرجة فقط (أزرار، مؤشرات، تقدم).
- **شكل ناعم**: زوايا مستديرة (`border-radius` متوسط إلى كبير) لإحساس بالمرونة وليس الصرامة.

### 1.3 اسم العلامة واستخدامه

| العنصر | الاستخدام |
|--------|-----------|
| **الاسم الكامل** | VibeFit AI Platform |
| **الاسم المختصر في الواجهة** | VibeFit |
| **الشعار النصي (Wordmark)** | `VibeFit` بخط Display؛ كلمة `AI` بلون ثانوي أخف إن وُجدت |
| **الأيقونة (مستقبلًا)** | شكل هندسي بسيط يوحي بالحركة/التقدم (سهم صاعد أو موجة) — خارج نطاق MVP |

### 1.4 ما نتجنبه بصريًا

- صور أجساد مثالية مبالغ فيها أو محتوى «قبل/بعد».
- ألوان حمراء عدوانية كخلفيات رئيسية.
- ازدحام بصري أو لوحات معلومات معقدة.
- لغة بصرية طبية (صليب أحمر، مخططات سريرية).

### 4.2 قواعد Typography

- **العناوين**: `neutral-800` أو `neutral-900`.
- **النص الأساسي**: `neutral-600`.

- نسبة الالتزام والإحصائيات: `text-h1` أو `text-h2` بوزن **700** بلون `primary-600` أو `accent-500`.

## 7. RTL وMobile-first

### 7.1 مبادئ Mobile-first

1. **التصميم يبدأ من 320px** ثم يتوسع.
2. **عمود واحد** افتراضيًا؛ الشبكات متعددة الأعمدة من `md` فما فوق.
3. **أزرار CTA بعرض كامل** على الجوال.
4. **تباعد أكبر بين العناصر القابلة للمس** (8px+ بين أزرار متجاورة).
5. **تقليل النصوص** في الشاشات الصغيرة — عناوين أقصر، فقرات 2–3 أسطر.

### 7.2 قواعد RTL

| العنصر | السلوك |
|--------|--------|
| اتجاه الصفحة | `dir="rtl"` على `<html>` |
| المحاذاة | نصوص عربية: يمين؛ أرقام وإحصائيات: يمكن وسط |
| الأيقونات الاتجاهية | تُعكس (أسهم، chevrons) |
| الأيقونات غير الاتجاهية | لا تُعكس (بحث، إغلاق، مستخدم) |
| الحقول LTR | البريد وكلمة المرور: `dir="ltr"` + محاذاة لليمين داخل الحقل |
| Navbar | الشعار يمين؛ عناصر التنقل يسار |
| Padding منطقي | استخدام `padding-inline-start/end` بدل left/right عند التحويل لـ CSS |

### 7.3 Breakpoints (للتحويل لاحقًا)

| Token | العرض |
|-------|-------|
| `screen-sm` | 640px |
| `screen-md` | 768px |
| `screen-lg` | 1024px |
| `screen-xl` | 1280px |

## 8. التنقل (Navigation)

### 8.1 الجوال (Mobile)

**النمط:** شريط علوي ثابت (Top Bar) + قائمة منسدلة (Hamburger).

```
┌──────────────────────────────────┐
│  [☰]          VibeFit      [ ]   │  ← ارتفاع 56px
├──────────────────────────────────┤
│                                  │
│         محتوى الصفحة             │
│                                  │
└──────────────────────────────────┘
```

| العنصر | الوصف |
|--------|--------|
| **الشعار** | وسط أو يمين (RTL) |
| **Hamburger** | يسار الشاشة — يفتح Drawer من الجانب الأيسر |
| **Drawer** | روابط عمودية: لوحة التحكم، التقييم، المتابعة، تسجيل الخروج |
| **صفحات عامة** | Landing/Login/Signup: روابط بسيطة («تسجيل الدخول» / «ابدأ») |

**مستخدم مسجّل على الجوال:**  
- يُفضَّل **شريط سفلي ثابت (Bottom Tab Bar)** للوصول السريع:

| Tab | الأيقونة | الوجهة |
|-----|----------|--------|
| الرئيسية | home | `/dashboard` |
| التقييم | clipboard | `/assessment` |
| المتابعة | calendar | `/checkin` |
| الحساب | user | قائمة: خروج |

ارتفاع Tab Bar: **64px** + safe-area-inset.

### 8.2 الكمبيوتر (Desktop)

**النمط:** شريط علوي أفقي (Horizontal Navbar).

```
┌────────────────────────────────────────────────────────┐
│  VibeFit          لوحة التحكم  التقييم  المتابعة    [خروج] │
└────────────────────────────────────────────────────────┘
```

| العنصر | الوصف |
|--------|--------|
| **الشعار** | يمين (RTL) — رابط للـ Dashboard أو Landing |
| **روابط التنقل** | وسط أو بعد الشعار — حالة active بخط سفلي `primary-500` |
| **خروج** | يسار — زر Ghost أو Secondary |
| **صفحات Auth** | شعار فقط + رابط «لديك حساب؟» |

**لا Sidebar دائم** في MVP — البساطة أولًا.

## 9. وصف بصري للصفحات (Page Visual Specs)

### 9.1 Landing Page (`/`)

**الهدف البصري:** إقناع الزائر بالبدء خلال 5 ثوانٍ.

```
┌─────────────────────────────────────────┐
│  Navbar: شعار + تسجيل الدخول + [ابدأ]  │
├─────────────────────────────────────────┤
│                                         │
│     عنوان Display: ابدأ رحلتك الرياضية │
│     فقرة body-lg بلون neutral-600       │
│                                         │
│     [ ابدأ مجانًا — Primary lg ]        │
│     [ تعرّف أكثر — Ghost ]              │
│                                         │
│  ┌─────┐ ┌─────┐ ┌─────┐               │
│  │خطوة1│ │خطوة2│ │خطوة3│  ← 3 بطاقات  │
│  └─────┘ └─────┘ └─────┘               │
│                                         │
│  قسم «كيف يعمل»: تقييم → توصية → متابعة │
│                                         │
│  Footer: إخلاء مسؤولية + حقوق          │
└─────────────────────────────────────────┘
```

| المنطقة | التفاصيل |

## 11. Accessibility والتباين (A11y)

### 11.1 التباين (Contrast)

| الزوج | النسبة المستهدفة | WCAG |
|-------|------------------|------|
| `neutral-800` على `neutral-0` | ≥ 12:1 | AAA |
| `neutral-600` على `neutral-0` | ≥ 7:1 | AAA للنص body |
| `neutral-0` على `primary-500` | ≥ 4.5:1 | AA للأزرار |
| `primary-600` على `primary-50` | ≥ 4.5:1 | AA للروابط |
| `error-700` على `error-50` | ≥ 4.5:1 | AA |

**قاعدة:** لا نص أساسي بـ `neutral-400` أو أفتح على خلفية فاتحة.

### 11.2 إمكانية الوصول

| المتطلب | التطبيق |
|---------|---------|
| **Focus visible** | حلقة `shadow-focus` على كل عنصر تفاعلي |
| **Labels** | كل حقل له `<label>` أو `aria-label` |
| **أخطاء الحقول** | `aria-invalid="true"` + `aria-describedby` للرسالة |
| **الأزرار المعطلة** | `disabled` + `aria-disabled` |
| **Loading** | `aria-busy="true"` على النموذج |
| **التباين للحركة** | احترام `prefers-reduced-motion` — إيقاف رسوم Skeleton المتحركة |
| **حجم اللمس** | 44×44px كحد أدنى |
| **تسلسل العناوين** | h1 واحد لكل صفحة؛ h2 للأقسام |
| **اللغة** | `lang="ar"` على HTML؛ `lang="en"` لقطع إنجليزية |

### 11.3 قارئ الشاشة

- نسبة الالتزام: `aria-label="نسبة الالتزام 60 بالمئة"`.
- شريط التقدم: `role="progressbar"` مع `aria-valuenow`.
