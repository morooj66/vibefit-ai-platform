## المتطلبات الوظيفية (Functional Requirements)

### FR-01: إنشاء الحساب

- النظام يقبل: البريد الإلكتروني، كلمة مرور (حد أدنى 8 أحرف).
- النظام يرفض البريد المكرر.
- بعد النجاح: إنشاء `profile` مرتبط بـ `user_id`.

### FR-02: تسجيل الدخول والخروج

- المصادقة عبر البريد وكلمة المرور.
- الجلسة تبقى نشطة وفق إعدادات Auth Provider (مثلاً Supabase Auth).
- تسجيل الخروج ينهي الجلسة.

### FR-03: نموذج التقييم الرياضي

الحقول الإلزامية (اقتراح MVP):

| الحقل | النوع | ملاحظات |
|-------|-------|---------|
| العمر | رقم (16–80) | |
| الجنس | اختيار | ذكر / أنثى / أفضل عدم الإفصاح |
| الطول (سم) | رقم | |
| الوزن (كغ) | رقم | |
| مستوى النشاط الحالي | اختيار | منخفض / متوسط / نشط |
| الهدف الرئيسي | اختيار | فقدان وزن / بناء عضلات / لياقة عامة / تحسين التحمل |
| أيام التمرين المتاحة أسبوعيًا | رقم (1–7) | |
| مدة الجلسة المفضلة (دقيقة) | اختيار | 20 / 30 / 45 / 60 |
| قيود أو إصابات (اختياري) | نص قصير | تنبيه: ليست استشارة طبية |

- التحقق من صحة الحقول قبل الإرسال (client + server).
- حفظ التقييم في `assessments` مع `user_id` و`created_at`.

### FR-04: توليد التوصية الأولية

- بعد حفظ التقييم، النظام يولّد توصية **منظمة** (أقسام ثابتة) بناءً على **قواعد محددة مسبقًا** (rule-based).
- أقسام التوصية المقترحة:
  - **ملخص الحالة**
  - **الهدف المقترح**
  - **تواتر التمرين المقترح**
  - **أنواع التمارين العامة**
  - **نصائح أمان عامة**
- حفظ التوصية في `recommendations` مرتبطة بـ `assessment_id`.
- توصية واحدة نشطة (active) لكل تقييم؛ عند تقييم جديد → توصية جديدة.

### FR-05: لوحة التحكم (Dashboard)

- تعرض للمستخدم المسجّل:
  - التوصية الحالية (أو رسالة "أكمل التقييم" إن لم يوجد).
  - نسبة الالتزام الأخيرة.
  - رابط/زر للمتابعة الأسبوعية.
  - ملخص آخر تقييم (اختياري في MVP).

### FR-06: المتابعة الأسبوعية

الحقول الإلزامية:

| الحقل | النوع |
|-------|-------|
| عدد أيام التمرين الفعلية | رقم (0–7) |
| الالتزام بالخطة (1–5) | مقياس |
| مستوى الطاقة (1–5) | مقياس |
| ملاحظات (اختياري) | نص |

- **قيد**: متابعة واحدة لكل أسبوع تقويمي (ISO week) لكل مستخدم.
- حفظ في `weekly_checkins`.

### FR-07: حساب نسبة الالتزام

**صيغة MVP المقترحة:**

```
نسبة الالتزام = (عدد أيام التمرين الفعلية / أيام التمرين المخططة من التقييم) × 100
```

- النتيجة تُقرب لأقرب عدد صحيح وتُحدّد بحد أقصى 100%.
- تُخزَّن أو تُحسب عند العرض من آخر `weekly_checkin` + بيانات التقييم النشط.
- تسجيل حدث في `activity_events` عند إرسال متابعة.

## معايير القبول (Acceptance Criteria)

### AC-01: التسجيل

- [ ] إدخال بريد صالح وكلمة مرور ≥ 8 أحرف → إنشاء حساب وإعادة توجيه لـ `/assessment`.
- [ ] بريد مكرر → رسالة خطأ واضحة بالعربية.
- [ ] حقول فارغة → رسائل تحقق على مستوى الحقل.

### AC-02: تسجيل الدخول

- [ ] بيانات صحيحة → جلسة نشطة + توجيه حسب حالة التقييم.
- [ ] بيانات خاطئة → رسالة "البريد أو كلمة المرور غير صحيحة" دون كشف أي منهما.

### AC-03: التقييم

- [ ] إرسال نموذج كامل صالح → حفظ + توليد توصية + توجيه لـ `/dashboard`.
- [ ] قيم خارج النطاق (مثلاً عمر 5) → رفض مع رسالة تحقق.

### AC-04: التوصية

- [ ] بعد التقييم تظهر التوصية بجميع الأقسام الخمسة المحددة.
- [ ] التوصية مرتبطة بالمستخدم الحالي فقط.

### AC-05: لوحة التحكم

- [ ] مستخدم بدون تقييم يرى دعوة لإكمال التقييم.
- [ ] مستخدم بتقييم يرى التوصية ونسبة الالتزام.

### AC-06: المتابعة الأسبوعية

- [ ] إرسال متابعة صالحة → حفظ + تحديث نسبة الالتزام + رسالة نجاح.
- [ ] محاولة متابعة ثانية لنفس الأسبوع → رفض مع رسالة "سبق إرسال متابعة هذا الأسبوع".

### AC-07: نسبة الالتزام

- [ ] 3 أيام فعلية من 5 مخططة → 60%.
- [ ] 0 أيام → 0%.
- [ ] أيام فعلية &gt; مخططة → 100% (حد أقصى).

## حالات الخطأ (Error Cases)

| السيناريو | السلوك المتوقع | رسالة للمستخدم (مثال) |
|-----------|----------------|------------------------|
| انقطاع الشبكة أثناء الإرسال | إبقاء البيانات في النموذج، زر إعادة المحاولة | تعذّر الاتصال. تحقق من الإنترنت وحاول مرة أخرى. |
| جلسة منتهية | إعادة توجيه لـ `/login` | انتهت جلستك. سجّل الدخول مرة أخرى. |
| خطأ خادم (5xx) | عدم فقدان البيانات المحلية للنموذج | حدث خطأ. حاول لاحقًا. |
| تقييم بدون حقول إلزامية | منع الإرسال | يرجى تعبئة جميع الحقول المطلوبة. |
| وصول غير مصرح لصفحة محمية | إعادة توجيه لـ `/login` | — |
| محاولة الوصول لبيانات مستخدم آخر | رفض (403/empty) | لا تملك صلاحية عرض هذا المحتوى. |
| فشل توليد التوصية بعد حفظ التقييم | التقييم محفوظ؛ إظهار حالة "جاري التوليد" أو إعادة محاولة | تعذّر توليد التوصية. أعد المحاولة من لوحة التحكم. |

## متطلبات الأمان (Security Requirements)

| ID | المتطلب |
|----|---------|
| SEC-01 | كلمات المرور لا تُخزَّن بنص صريح؛ تُدار عبر Auth Provider |
| SEC-02 | جميع الاتصالات عبر HTTPS في الإنتاج |
| SEC-03 | **Row Level Security (RLS)**: كل مستخدم يقرأ/يكتب بياناته فقط |
| SEC-04 | التحقق من صحة المدخلات على الخادم (ليس client فقط) |
| SEC-05 | حماية من CSRF وفق إطار العمل المستخدم |
| SEC-06 | Rate limiting على نقاط تسجيل الدخول وإنشاء الحساب |
| SEC-07 | عدم تضمين بيانات حساسة في `activity_events` (لا كلمات مرور، لا تفاصيل طبية خام إن أمكن تلخيصها) |
| SEC-08 | تسجيل الخروج يبطل الجلسة/التوكن |
| SEC-09 | رسائل الخطأ لا تكشف وجود حساب من عدمه عند تسجيل الدخول (رسالة موحّدة) |

## قيود وإخلاء مسؤولية (ضمن المتطلبات)

- المنصة **لا تقدم تشخيصًا أو علاجًا طبيًا**.
- حقل "قيود أو إصابات" للمعلومات العامة فقط؛ التوصية تتضمن تنبيهًا باستشارة مختص عند الحاجة.
- التوصية **إرشادية عامة** وليست بديلاً عن مدرب أو طبيب.

### FR-F01: العمر الصحي التقديري (Estimated Wellness Age)

#### التعريف

مؤشر **توعوي غير طبي** — ليس تشخيصًا ولا تقييمًا سريريًا. لا يقيس الشيخوخة الخلوية أو الجينية. يُعرض كنطاق تقريبي ودرجة، لا كرقم مؤكد.

#### المدخلات المحتملة

| المدخل | المصدر | ملاحظات |
|--------|--------|---------|
| `chronological_age` | آخر `assessment` | العمر الزمني |
| `activity_level` | آخر `assessment` | منخفض / متوسط / نشط |
| `weekly_training_frequency` | `weekly_checkins` + `assessment` | متوسط أيام التمرين |
| `average_sleep_hours` | حقل مستقبلي أو `input_snapshot` | اختياري — يُجمع فقط عند الحاجة |
| `sedentary_hours` | حقل مستقبلي أو `input_snapshot` | اختياري |
| `energy_level` | `weekly_checkins` | متوسط مستوى الطاقة |
| `adherence_history` | `weekly_checkins` | سجل نسب الالتزام |
| `optional_fitness_indicators` | `assessment` / حقول اختيارية | مؤشرات لياقة عامة — غير طبية |

> **قيد البيانات:** لا تُستخدم بيانات شخصية أو طبية غير ضرورية. لا تشخيصات، لا سجلات طبية، لا بيانات جينية.

#### المخرجات

| المخرج | النوع | الوصف |
|--------|-------|--------|
| `chronological_age` | `int` | العمر الزمني المستخدم في الحساب |
| `estimated_wellness_age_min` | `int` | الحد الأدنى للنطاق التقديري |
| `estimated_wellness_age_max` | `int` | الحد الأعلى للنطاق التقديري |
| `wellness_score` | `int` | 0–100 — درجة صحية عامة توعوية |
| `influencing_factors` | `array` | عوامل مؤثرة قابلة للتفسير (مثلاً: انخفاض النشاط، قلة النوم) |
| `disclaimer` | `text` | تنبيه بأن النتيجة تقديرية وغير طبية |
| `calculated_at` | `timestamptz` | وقت الحساب |
| `model_version` | `text` | إصدار النموذج/الخوارزمية |

#### معايير القبول (AC-F01)

- [ ] النتيجة تظهر **كنطاق** (`estimated_wellness_age_min` – `estimated_wellness_age_max`) وليس رقمًا واحدًا مؤكدًا.
- [ ] يظهر **تنبيه واضح** بأنها تقديرية وغير طبية (عبر `disclaimer`).
- [ ] تُعرض **العوامل المؤثرة** (`influencing_factors`) بلغة مفهومة للمستخدم.
- [ ] يمكن للمستخدم رؤية **تاريخ التقديرات** السابقة.
- [ ] لا يُصدر النظام **تشخيصًا** أو **توصيات علاجية**.
- [ ] لا يُستخدم مصطلح «العمر البيولوجي الحقيقي» في أي مكان في الواجهة أو الوثائق.
- [ ] عند عدم توفر بيانات كافية → رسالة «لا توجد بيانات كافية لإجراء التقدير حاليًا».

#### حالات الخطأ

| السيناريو | السلوك |
|-----------|--------|
| بيانات غير كافية | عدم إظهار رقم؛ رسالة توضيحية + دعوة لإكمال المتابعات |
| فشل الحساب | عدم عرض نتيجة جزئية؛ تسجيل الخطأ في السجلات |

### FR-F02: توقع انخفاض الالتزام (Adherence & Drop-off Risk Prediction)

#### التعريف

تقدير **احتمالي** لمستوى مخاطرة انخفاض الالتزام أو التوقف. ليس حكمًا على سلوك المستخدم.

#### المدخلات المحتملة

| المدخل | المصدر | ملاحظات |
|--------|--------|---------|
| `planned_sessions` | آخر `assessment` | أيام التمرين المخططة أسبوعيًا |
| `completed_sessions` | `weekly_checkins` | أيام التمرين الفعلية |
| `previous_adherence_rate` | `weekly_checkins` | متوسط أو آخر نسبة التزام |
| `missed_checkins` | `weekly_checkins` | عدد الأسابيع بدون متابعة |
| `energy_level` | آخر `weekly_checkin` | مقياس 1–5 |
| `difficulty_level` | حقل مستقبلي في المتابعة (اختياري) | إن وُجد — غير طبي |
| `days_since_last_activity` | `activity_events` / `weekly_checkins` | أيام منذ آخر نشاط مسجّل |
| `training_days` | `assessment` | الأيام المتاحة للتمرين |
| `experience_level` | `assessment.activity_level` | مستوى الخبرة/النشاط |

> **قيد البيانات:** مدخلات مشتقة من سلوك المنصة فقط — دون بيانات شخصية أو طبية غير ضرورية.

#### المخرجات

| المخرج | النوع | الوصف |
|--------|-------|--------|
| `risk_level` | `enum` | `low` / `medium` / `high` |
| `probability` | `numeric` | احتمال تقريبي (مثلاً 0.0–1.0 أو نسبة مئوية مع تسمية «تقريبي») |
| `main_factors` | `array` | أهم العوامل المؤثرة قابلة للتفسير |
| `suggested_action` | `text` | إجراء مبسط (تذكير، تقليل الأيام، إعادة تقييم) — **اقتراح فقط** |
| `model_version` | `text` | إصدار النموذج |
| `predicted_at` | `timestamptz` | وقت التوقع |

#### معايير القبول (AC-F02)

- [ ] يُعرض صراحةً أن النتيجة **احتمالية وليست مؤكدة**.
- [ ] يوضح **أهم العوامل المؤثرة** (`main_factors`) بلغة محايدة وداعمة.
- [ ] **لا يصف** المستخدم بأنه فاشل أو غير ملتزم.
- [ ] **لا يغيّر** خطة التمرين أو التوصية **تلقائيًا** دون موافقة المستخدم.
- [ ] يمكن استخدام النتيجة لتشغيل **تذكير** أو **اقتراح مبسط** في Dashboard.
